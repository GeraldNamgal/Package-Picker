How It Works

Modern applications rely heavily on Open-Source-Software packages. But how do you find the good ones? Search engines only know what you ask them, and humans can't follow the millions of packages available. Package Picker is here to help.

Package Picker looks at the dependencies in your application and feeds these into a machine learning algorithm, which recommends packages that are tailored to your specific application.

How Do We Calculate the Package Picker Score?

The Package Picker score is calculated by taking into account several metrics. It uses a weighted average of four key scores: Used-together score, collaborative-filtering score, Percent change trending score, absolute change trending score. Each one of these scores is defined on a scale of 1 to 10 with 10 being the best.

What is the Used-Together Score?

It is how often packages appear in the same dependency list. e.g. in Javascript projects, this would be how often two packages appear in the same package.json file.

What is the Collaborative Filtering Score?

This is the similarity score. Given you use Package A, this score calculates the probability that you will like and use Package B.

What is the Absolute Change Trending Score?

This score captures how much a package has increases in use since this time last year. This score takes difference between the download count of the package today and the download count of the package one year ago in the absolute count. Then this difference is compared to all the other packages in the database and fitted to a bell-curve distribution. Think of this score as the popularity-contest score, the packages with the most increase in downloads since this time last year will be rewards by this score.

What is the Relative Change Trending Score?

This score is similar to the Absolute Change Trending Score, but here it compares the package in terms of percent growth. It measures the percentage growth of this package. It takes the number of downloads today divided by the number of downloads a year ago. Then this difference is compared to all the other packages in the database and fitted to a bell-curve distribution. Think of this score as the niche popularity-contest score e.g. a small bio-medical software package that has gained 1,000% growth in use since Coronavirus won't be overshadowed, by some common StringUtils package. This allows more niche packages to get the recognition they deserve.
